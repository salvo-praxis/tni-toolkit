Page Layout Patterns

Standardized Header

All pages use consistent header with colored title and optional back button.

.header { text-align: center; padding: 40px 0 30px; border-bottom: 1px solid #30363d; margin-bottom: 30px; } h1 { color: #00ff88; text-shadow: 0 0 20px rgba(0, 255, 136, 0.3); margin: 0 0 8px 0; font-size: 20px; } h1 span { color: #58a6ff; }

Standardized Footer

Three-row footer with links, tagline, and version/license badges.

/* Row 1: Links */ GitHub | Contributions Log | [page links] | TNI on Steam /* Row 2: Tagline */ .footer-tagline { color: #6e7681; font-size: 10px; } /* Row 3: Badges */ .version-badge { background: rgba(0, 255, 136, 0.1); } .license-badge { background: rgba(88, 166, 255, 0.1); }

Best Practices

DO ‚úì

  • Use design tokens for colors/spacing (enables future theming)
  • Follow the existing panel structure for consistency
  • Add transitions to interactive elements
  • Use semantic HTML and proper heading hierarchy
  • Test at mobile breakpoints (<600px)
  • Include descriptive title attributes on interactive elements

DON'T ‚úó

  • Use inline styles ‚Äî keep CSS in the style block
  • Add external dependencies (CDNs, libraries)
  • Use px for everything ‚Äî relative units for spacing
  • Forget the box-sizing reset
  • Use bright white (#fff) ‚Äî it's too harsh
