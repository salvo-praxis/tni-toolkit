Data File Guidelines

JSON Structure

All data files use this header format in _meta:

{
  "_meta": {
    "game": "Tower Networking Inc.",
    "dataset": "dataset-name",
    "version": "1.0.0",
    "last_updated": "YYYY-MM-DD",
    "description": "What this dataset contains",
    "sources": [...],
    "contributors": [...],
    "corrections": [...],
    "future_additions": [...]
  }
}

When Editing Data Files

Maintain structure â€” Don't change field names or nesting
Update metadata â€” Bump version, update date, add yourself to contributors
Add sources â€” Credit where data came from
Log corrections â€” Add to corrections array with version prefix
Track suggestions â€” Use future_additions for ideas not yet implemented
